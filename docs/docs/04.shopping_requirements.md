## 📝 고객 요구사항 정리
1. 페이지 연동: 상품 목록, 장바구니, 결제 등 모든 페이지에서 자연스러운 이동.
2. 데이터 일관성: 페이지 간 이동이 있어도 데이터 일관되게 유지. 상품 목록에서 장바구니에 강품을 담았을 때, 다른 페이지에서도 그 상품이 장바구니에 담겨 있음을 명확히 보여주는 것.
3. 상세 페이지 추가: 상세페이지에서는 상품에 대한 상세 정보와 관련 상품 등을 보여주기. 관련 상품은 같은 브랜드의 상품.
4. 테스트 URL 제공 

## 📋 기능 목록
ProductListPage (상품 목록 페이지)
기능: 전체 상품 목록을 보여줌
상품 클릭 시: ProductDetailPage로 이동
“장바구니 담기” 클릭 시 Global Cart Context에 상품 추가
데이터 일관성 유지: 다른 페이지로 가도 장바구니 수량 그대로 유지
헤더에서 “장바구니”, “결제 페이지”로 이동 가능 (페이지 연동)

2. ProductDetailPage (상품 상세 페이지)
기능: 상품의 이미지, 설명, 가격, 브랜드, 재고 정보 표시
장바구니 담기 기능 포함
관련 상품 리스트 표시 (같은 브랜드 상품)
기존 상품의 brand 값을 기준으로 필터링
유저 클릭 시 해당 상품 상세 페이지로 이동

3. CartPage (장바구니 페이지)
기능: 현재 장바구니에 담긴 상품 목록 표시
수량 변경, 삭제 기능
Global Cart Context 기반 → 페이지 이동해도 데이터 유지
“결제하기” 버튼 클릭 시 CheckoutPage로 이동

4. CheckoutPage (결제 페이지)
기능: 카드 정보 입력
배송 정보 입력
결제 버튼 클릭 시 결제 프로세스 완료
완료 시 OrderCompletePage로 이동

5. OrderCompletePage (결제 완료 페이지)
기능: “결제가 정상적으로 완료되었습니다” 메시지
주문 번호 표시
구매한 상품 리스트 표시
“상품 목록으로 돌아가기” / “주문내역 보기” 버튼 제공
Global Cart Context 초기화 (결제 후 장바구니 비움)

6. Header / NavigationBar (페이지 연동 핵심 컴포넌트)
기능: ProductListPage, CartPage, CheckoutPage 등으로 이동 가능
페이지 간 자연스러운 이동 보장
